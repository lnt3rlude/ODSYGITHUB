# 📝 REST API для керування сутностями (ЛР1) — In-Memory Backend

Цей проєкт є бекенд-частиною веб-застосунку для автоматизації бізнес-процесів (керування користувачами, категоріями, товарами та замовленнями). Відповідно до вимог технічного завдання (ТЗ), архітектуру застосунку повністю переведено на **In-Memory архітектуру** (робота виключно в оперативній пам'яті сервера) без використання зовнішніх баз даних чи SQL-рушіїв.

Проєкт реалізовано на базі сучасного Node.js стек-технологій із чітким розділенням на архітектурні шари (Routes -> Controllers -> Services -> Repositories).

## ✨ Основний функціонал (CRUD)

Репозиторій надає повноцінне REST API для сутностей системи, зокрема для обов'язкової сутності **Користувачів (`Users`)** та **Категорій (`Categories`)**:
- **Створення (Create):** `POST` запити з автоматичною генерацією UUID на стороні сервера та валідацією вхідних DTO.
- **Перегляд (Read):** `GET` запити для отримання повного списку сутностей або пошуку конкретного запису за його ID.
- **Редагування (Update):** `PUT`/`PATCH` запити для часткового чи повного оновлення полів об'єкта.
- **Видалення (Delete):** `DELETE` запити для надійного видалення об'єктів з оперативної пам'яті зі збереженням REST-специфікації (повернення статусу `204 No Content`).

## 🔍 Технічні особливості архітектури

- **In-Memory Storage:** Дані зберігаються в статичних масивах у межах життєвого циклу процесу Node.js. Операції пошуку та фільтрації реалізовані через оптимізовані методи JavaScript (`find`, `filter`, `findIndex`, `splice`).
- **Асинхронний інтерфейс:** Усі методи репозиторіїв та сервісів повертають `Promise` (`async/await`), що гарантує легку міграцію на повноцінну БД у майбутньому без зміни сигнатур методів.
- **Уніфікована обробка помилок:** Централізований `globalErrorHandler` перехоплює всі винятки, логує їх залежно від середовища (`NODE_ENV`) та повертає клієнту стандартну відповідь за ТЗ: `{ "error": { "code": "...", "message": "...", "details": [...] } }`.
- **Строга валідація даних:** Перед обробкою запиту сервіси перевіряють типи даних, обов'язковість полів та обмеження довжини рядків за допомогою кастомних валідаторів без залучення важких сторонніх бібліотек.

## 🛠 Використані технології

- **Node.js (v24+):** Середовище виконання JavaScript на сервері.
- **TypeScript (v6+):** Строга типізація коду, інтерфейсів та DTO.
- **Express (v5+):** Потужний мінімалістичний HTTP-фреймворк для побудови REST API.
- **TSX (TypeScript Execute):** Сучасний, надшвидкий інструмент розробки для запуску та автоматичного перезапуску додатка (`watch` режим) на macOS/Windows.
- **Cors:** Захист та налаштування крос-доменних запитів.
