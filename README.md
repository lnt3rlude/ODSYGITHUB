# 📝 REST API та Інтеграція Проєкту (ЛР4) — Secure Full-Stack SQLite Backend

Цей проєкт є фінальною інтеграційною частиною веб-застосунку для автоматизації бізнес-процесів (керування користувачами, категоріями, товарами та замовленнями). Відповідно до вимог технічного завдання (ТЗ) ЛР4, початкову In-Memory архітектуру було повністю замінено на реляційну базу даних **SQLite**, а серверний код інтегровано з клієнтською частиною (`odsy.frontend`) в єдину екосистему розробки (Workspace).

Особливу увагу в цій ревізії приділено **безпеці даних (Security-First Approach)**: архітектуру додатку спроєктовано так, щоб повністю нівелювати критичні вразливості OWASP Top 10, такі як SQL-ін'єкції, XSS та IDOR атаки.

Проєкт реалізовано на базі сучасного Node.js стек-технологій із чітким розділенням на архітектурні шари (Routes -> Middlewares -> Controllers -> Services -> Repositories -> DB Client).

## ✨ Нове в версії 0.4.0 (Full-Stack Інтеграція & Безпека)

- **Workspace Routing:** Налаштовано повну синхронізацію між клієнтом та сервером. Вирішено проблеми відносних шляхів та налаштовано роботу в режимі монорепозиторію.
- **Уніфіковані API Префікси:** Усі маршрути бекенду приведені до єдиного стандарту `/api/...` (наприклад, `/api/users`), що дозволило фронтенду стабільно виконувати HTTP-запити без помилок маршрутизації (404 Not Found).
- **Повноцінний SQLite CRUD:** Операції створення, читання, оновлення та видалення сутностей повністю переведено з оперативної пам'яті на реальні SQL-запити до файлу бази даних `app.db`.
- **Конфігурація CORS:** Налаштовано безпечний обмін даними між різними портами фронтенду (Vite) та бекенду (Express) з чітким обмеженням дозволених джерел (Origin Whitelisting).

## 🛡 Захист від кібератак та безпека даних

Впроваджено комплексний захист на кожному рівні взаємодії клієнт-сервер-БД:

1. **Захист від SQL-ін'єкцій (SQL Injection Protection):**
   У репозиторіях повністю виключено динамічну конкатенацію SQL-рядків. Усі запити до SQLite реалізовано виключно через **параметризовані запити (Prepared Statements)** та плейсхолдери (`?` або `$variable`). Вхідні дані користувача ніколи не потрапляють у тіло SQL-команди напряму, що унеможливлює модифікацію логіки запитів зловмисниками.

2. **Захист від IDOR (Insecure Direct Object Reference):**
   Доступ до об'єктів (наприклад, перегляд чи видалення замовлень за `id`) захищено шаром сервісної валідації. Система не просто перевіряє існування сутності в БД, а й виконує **строгу перевірку контексту прав доступу (Access Control Verification)**. Користувач може отримати доступ лише до тих записів, які належать безпосередньо йому або на які він має підтверджені права, що блокує спроби перебору `id` у URL.

3. **Захист від XSS (Cross-Site Scripting):**
   - На рівні бекенду впроваджено **санітизацію вхідних даних (Input Sanitization)**. Усі текстові поля, що приходять у DTO, проходять очищення від потенційно небезпечних HTML-тегів та JavaScript-скриптів.
   - Налаштовано спеціальні HTTP-заголовки безпеки (Content Security Policy, X-XSS-Protection).
   - На фронтенді виведення даних ізольовано за допомогою вбудованих механізмів екранування фреймворку, що запобігає виконанню шкідливого коду в браузері користувача.

## 🔍 Технічні особливості архітектури

- **Реляційне збереження (SQLite3):** Дані надійно зберігаються у локальному файлі бази даних за допомогою асинхронних методів (`db.run`, `db.get`, `db.all`). Дані більше не зникають після перезапуску сервера.
- **Автоматичні міграції та Сідінг:** Реалізовано ізольований механізм розгортання структури таблиць за допомогою SQL-скриптів та автоматичного наповнення бази первинними тестовими даними (`seed.ts`).
- **Асинхронний інтерфейс:** Усі шари додатка використовують синтаксис `async/await` та промісифіковані запити до БД, що гарантує максимальну швидкість обробки запитів від клієнта.
- **Уніфікована обробка помилок та валідація:** Валідатори DTO перевіряють типи даних на рівні Express-middleware, а централізований `globalErrorHandler` перехоплює всі винятки (включаючи порушення унікальності SQL констрейнтів) та повертає клієнту стандартний JSON за ТЗ, **не розкриваючи системні деталі (stack trace)** зловмисникам.

## 🛠 Використані технології

- **Node.js (v24+):** Серверне середовище розробки та виконання JavaScript.
- **TypeScript (v6+):** Наскрізна сувора типізація інтерфейсів, DTO та моделей даних.
- **Express (v5+):** Потужний HTTP-фреймворк для побудови REST API з підтримкою маршрутизації та middleware-безпеки.
- **SQLite3:** Легковаговий SQL-рушій для збереження даних у файл додатка.
- **Vite:** Сучасний інструмент збірки та гарячого перезапуску фронтенд-частини додатка.
- **TSX (TypeScript Execute):** Оптимізований інструмент для швидкого запуску сервера в `watch` режимі.
